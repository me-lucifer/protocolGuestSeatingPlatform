# **App Name**: Protocol Guest Seating Platform

## Core Features:

- Role Selection Landing Page: Full-screen layout with role cards for 'Super Admin', 'Protocol Admin', 'Protocol Officer', and 'Guest'.
- Super Admin Dashboard: Dashboard for IT admins to manage platform settings and configurations. This is a front-end prototype only and NO real backend.
- Protocol Admin Dashboard: Event managers can configure and manage guest seating events. This is a front-end prototype only and NO real backend.
- Protocol Officer Interface: Entrance agents use this interface to check-in guests. This is a front-end prototype only and NO real backend.
- Guest View: A simple interface for guests to view event details. This is a front-end prototype only and NO real backend.
- Persistent Navigation: A 'Back to Role Selection' link in the top-left on all inner pages, and a top bar displaying the app logo/title on the left and current role on the right.
- Dummy Data: Use local state or hard-coded dummy JSON arrays for events, guests, seating, etc.

## Style Guidelines:

- Primary color: Deep navy blue (#191970) for a premium, government/diplomatic feel.
- Background color: Light gray (#F0F0F0), a neutral color with very low saturation and high brightness.
- Accent color: Muted gold (#B8860B), analogous to deep navy blue.
- Font: 'Inter' sans-serif font for modern, highly readable text.
- Use a central white card layout with rounded corners and soft shadows.
